## 2. 研究要素一覧

### 要素1: C-RADIO ViTバックボーン置換

**新規性**: ★★★★☆
**難易度**: ★★★☆☆
**インパクト**: ★★★★☆

**概要**: Qwen VLMの内蔵ViTをNVIDIA C-RADIOv4-SO400Mに置き換え、マルチティーチャー蒸留ViTがVLAの視覚表現としてどのように機能するかを検証する。

**根拠**:
- Theiaの発見：空間特徴エントロピーとロボット学習性能の相関 R=0.943
- C-RADIOはSigLIP2（意味的）+ DINOv3（空間）+ SAM3（セグメンテーション）を統合
- 412Mパラメータで個々の教師モデルを上回る性能

**技術的詳細**:
- C-RADIO出力dim: 1152、パッチサイズ: 16
- Qwen3-VL-4B入力dim: 2560、パッチサイズ: 16（互換）
- Projector設計: 2層MLP（1152 → GELU → 2560）+ Spatial Merge (2x2)
- DeepStack代替: C-RADIO中間層からのフック or 単一層出力での代替

**検証指標**:
- LIBERO成功率（ベースラインQwen ViT vs C-RADIO）
- 空間特徴エントロピーの測定と性能相関
- 推論レイテンシへの影響

### 要素2: MEMマルチスケールメモリ統合

**新規性**: ★★★☆☆（MEM自体は既存、Qwen統合が新規）
**難易度**: ★★★★☆
**インパクト**: ★★★★★

**概要**: PIのMEM（Multi-Scale Embodied Memory）をQwen VLMに統合し、短期ビデオメモリと長期言語メモリの二重経路メモリシステムを実装する。

**短期メモリ（ビデオエンコーダ）**:
- ViTに空間-時間分離アテンションを導入
- 4層ごとに因果的時間アテンション挿入
- 追加パラメータなし（正弦波時間位置エンコーディングで初期化）
- 上層で過去フレームのパッチ表現をドロップ → 現在フレームのみLLMに渡す
- 計算量: O(Kn² + nK²)（結合アテンションO(n²K²)より大幅削減）

**長期メモリ（言語ベース）**:
- 完了イベントの自然言語要約 m_t
- VLM自体がメモリの圧縮・更新を学習
- **成功時のみ更新**（ナイーブ連結は20-25%性能低下）
- 圧縮戦略：冗長な詳細を削除（例：個別色→「3つの椀」）

**Qwen統合のポイント**:
- Qwen2.5-VLは既にWindow + Full Attentionの混合 → MEMの分離アテンションと親和性高
- temporal_patch_size=2が既に時間的処理を持つ → MEM時間アテンションとの統合が自然
- 長期メモリはQwenの言語能力で生成 → 多言語メモリも可能

### 要素3: RLT（RL Tokens）低ポリシー精密制御

**新規性**: ★★★☆☆（RLT自体は既存、統合が新規）
**難易度**: ★★★☆☆
**インパクト**: ★★★★☆

**概要**: VLAの内部表現から情報ボトルネックとしてRLトークンを抽出し、小規模Actor-Criticで精密操作タスクを高速改善する。

**アーキテクチャ**:
```
VLA Internal Embeddings
        │
   ┌────▼─────┐
   │ Encoder   │ (RLトークン抽出)
   │ (小規模   │
   │ Transformer)│
   └────┬─────┘
        │
   RL Tokens (情報ボトルネック)
        │
   ┌────▼─────┐     ┌──────────┐
   │  Actor    │     │  Critic   │
   │ (小規模)  │     │ (小規模)  │
   └────┬─────┘     └──────────┘
        │
   Action Edit (VLA予測の「編集」)
        │
   Final Action
```

**特徴**:
- VLAの行動を「置換」ではなく「編集」する設計
- Reference-action dropout で単純コピー防止
- オフポリシーRL：毎秒数百回の更新が可能
- **実機上15分〜2時間で学習完了**

**期待される効果**:
- 精密接触タスク（ネジ締め、コネクタ挿入等）で4-8倍の性能改善
- 100Hz対応（小規模ネットワークの推論は~1ms）

### 要素4: RECAP高ポリシーRL改善

**新規性**: ★★★☆☆
**難易度**: ★★★★☆
**インパクト**: ★★★★☆

**概要**: pi0.6のRECAP法を高ポリシーに適用し、アドバンテージ条件付けでVLMの言語能力を保持しつつRL改善を行う。

**アドバンテージ条件付け**:
- テキスト入力に「Advantage: positive」「Advantage: negative」を追加
- VLMは良い行動も悪い行動もデータから学習
- 推論時に分類器フリーガイダンスで「良い行動」方向にガイド
- 数式: `π*(a|o,l) ∝ π_ref(a|o,l) × (π_ref(a|I,o,l) / π_ref(a|o,l))^β`

**価値関数**:
- 独立した小規模VLM（670Mパラメータ相当）
- 201個の離散化ビンによる分布的価値関数
- タスク完了までのステップ数を予測

**学習サイクル**:
1. 自律データ収集（オプション：専門家介入）
2. 価値関数学習
3. アドバンテージ条件付きポリシー抽出
4. 繰り返し

### 要素5: フローマッチング行動生成ヘッド

**新規性**: ★★☆☆☆（確立された手法）
**難易度**: ★★★☆☆
**インパクト**: ★★★★★

**概要**: pi0/Xiaomi-R0で実証されたフローマッチングベースの行動生成ヘッドを実装する。

**設計選択**:
- **DiT (Diffusion Transformer)** ベース（Xiaomi-R0方式）: 16層DiT
- **条件付け**: VLM最終層からのKVキャッシュ + 固有受容覚 + ノイズ付き行動トークン
- **デノイジングステップ**: 10ステップの前方オイラー法
- **アクションチャンク**: 6-20ステップの行動系列を一括予測

**Xiaomi-R0の革新を取り込むべき要素**:
- Lambda-Shape注意マスク（視覚・言語条件への注意促進）
- RoPE位置オフセット（ショートカット学習防止）
- 動的損失再重み付け（非同期学習時の品質向上）
- Attention Sinkトークン（学習安定化）

### 要素6: 学習パイプライン設計

**概要**: Pre-train → SFT → RL の3段階パラダイムに従う統合学習パイプライン。

```
Stage 0: VLM事前学習（既存重みを利用）
  │
Stage 1: Alignment Pre-training
  │ C-RADIO (frozen) → Projector (train) → LLM (frozen)
  │ データ: 画像-テキストペア + ビデオ-言語タスク
  │
Stage 2: Robot SFT
  │ C-RADIO (low-lr) → Projector (train) → LLM (train, LoRA)
  │ + MEM短期・長期メモリの学習
  │ + アクションエキスパートの学習
  │ データ: テレオペレーションデモ + OXE + DROID
  │
Stage 3: Online RL (RECAP)
  │ 高ポリシーのアドバンテージ条件付き改善
  │ データ: 自律走行 + 専門家修正
  │
Stage 4: RLT Fine-tuning
  │ 低ポリシーの精密制御改善
  │ データ: 実機上15分〜2時間
  │
Stage 5: (Optional) Test-time Adaptation
    テスト時RL適応
```

## 3. 競合との差別化

| 特徴 | Ours | pi0.6/MEM/RLT | Xiaomi-R0 | OpenVLA-OFT |
|------|------|---------------|-----------|-------------|
| マルチティーチャー蒸留ViT | **C-RADIO** | なし | Qwen ViT | DinoSigLIP |
| 長期言語メモリ | **MEM式** | MEM | なし | なし |
| 短期ビデオメモリ | **MEM式** | MEM | なし | なし |
| 高ポリシーRL | **RECAP** | RECAP | なし | なし |
| 低ポリシーRL | **RLT** | RLT | なし | なし |
| 統合システム | **All-in-one** | 個別 | 部分的 | 部分的 |
| オープンソース | **予定** | 非公開 | 公開 | 公開 |
| VLMベース | **Qwen3.5** | Gemma 3 | Qwen3-VL | Prismatic |

**核心的な差別化**: PIの技術（MEM+RLT+RECAP）を統合し、さらにC-RADIOという新しいビジョンバックボーンの有効性を検証する**初のオープンな統合システム**。加えて、**Qwen3.5-4B（Gated DeltaNet線形アテンション搭載）をVLAバックボーンに採用する初の研究**となる。

## 追加研究要素: Qwen3.5-4Bの採用とGated DeltaNet

### 要素7: Qwen3.5-4B — 線形アテンションVLMのVLA統合

**新規性**: ★★★★★
**難易度**: ★★★☆☆
**インパクト**: ★★★★★

**概要**: 2026年2月リリースのQwen3.5-4Bは、従来のフルアテンションVLMとは根本的に異なる**Gated DeltaNet（線形アテンション）** を採用。VLAバックボーンとしてQwen3.5を使用した研究は未発表であり、線形アテンションがロボット制御にどのような影響を与えるかの検証自体が研究貢献となる。

**Qwen3-VL-4Bからの主要変更点**:

| 特性 | Qwen3-VL-4B | Qwen3.5-4B |
|------|-------------|------------|
| アテンション | 全層フルアテンション | **3:1ハイブリッド** (DeltaNet:Full) |
| 計算量 | O(n²) | **O(n)** (線形アテンション層) |
| DeepStack | [5, 11, 17] | **廃止（空配列）** |
| vocab_size | 151,936 | **248,320** |
| MTP | なし | **あり** |
| 4Bでの性能 | ベースライン | **30Bクラスを超越** |
| コンテキスト | 262K | 262K **(最大1M+)** |

**VLA統合における利点**:
1. **推論速度**: 線形アテンションO(n)により、長い観測履歴・動画フレーム処理が高速
2. **DeepStack廃止**: ViT出力がシンプル → C-RADIO置換が容易
3. **MTP**: 投機的デコーディングでサブタスク指示の生成が高速化
4. **RNN的リカレント特性**: 逐次アクション生成に自然に適合
5. **CountBench 96.3%**: 物体カウント能力が高く、MEM長期メモリの状態追跡に直結

**検証すべき研究課題**:
- Gated DeltaNetのVLAファインチューニング時の学習安定性
- 線形アテンション層でのRECAPアドバンテージ条件伝播の挙動
- RLトークン抽出元としてのDeltaNet内部状態の適性
- フルアテンション比率（現在1/4）の最適化（VLA用途では異なる可能性）
