## 1. RLT: RL Tokens (Physical Intelligence)

### 1.1 概要

RLT (Reinforcement Learning Tokens) は、Physical Intelligence社が開発した手法であり、VLAモデルから**コンパクトな「RLトークン」を抽出**し、小規模なActor-Criticネットワークで高精度マニピュレーションタスクを高速に改善する手法である。フルモデルのファインチューニングを必要とせず、**実ロボット上で数時間（最短15分）のデータで学習可能**という特徴を持つ。

### 1.2 アーキテクチャ

**RLトークンの抽出プロセス:**
- VLAモデルに対し、エンコーダ・デコーダTransformerを追加し、再構成損失（reconstruction loss）で学習する
- RLトークンは**情報ボトルネック**として機能し、元のVLA埋め込みをデコーダが再構成できる最小限の情報を保持する
- これにより、タスクに最も関連する内部表現のみが要約される

**ポリシー構造:**
- 小規模なActorおよびCriticネットワークがRLトークン表現上で動作する
- Actorは、VLAの予測アクションを入力として受け取り、**アクションを「置き換える」のではなく「編集する」**ように学習する
- Reference-action dropoutにより、学習初期の単純コピーを防止する
- オプションとして人間の介入をRL更新ループに直接統合可能

**アクション空間:**
- VLAの時間的構造に合わせたアクションチャンク（action chunks）上で動作する
- ポリシー更新はリファレンスアクションに向けて正則化される

### 1.3 学習パイプライン

- オフポリシーRL手法を採用し、ロボット上で毎秒数百回の更新が可能
- VLAの行動が妥当な場合はそれに基づく探索を行い、Criticが改善を検出した場合のみ逸脱する
- ロボットハードウェア上で直接学習が実行される
- 15分のデータでも結果が得られ、リセット含め約2時間で完全な学習サイクルが完了する

### 1.4 報酬信号

論文では明示的な報酬関数の定式化は詳述されていないが、RLトークン学習には再構成損失、ポリシー学習にはCriticベースの改善シグナルが用いられている。

### 1.5 主要結果

| タスク | ベースモデル | RLT適用後 | 改善倍率 |
|--------|-------------|-----------|---------|
| M3ネジ締め | 1.7回/10分 | 14回/10分 | 8.2x |
| 結束バンド通し | 2.8回/10分 | 13回/10分 | 4.6x |
| イーサネットケーブル挿入 | 147回/10分 | 400回/10分 | 2.7x |
| 電源コード挿入 | 136回/10分 | 600回/10分 | 4.4x |

特筆すべき点として、イーサネットケーブル挿入タスクでは、RLTポリシーが**人間のテレオペレーション速度を上回った**（RLT試行の半分が、テレオペレーションのどのデモンストレーションよりも高速）。

### 1.6 重要な知見

- RLTは**最終精密フェーズ**（サブミリメートル精度が要求される接触リッチな瞬間）を特にターゲットとする
- ベースモデルは初期フェーズ（工具の把持など）には成功するが、精密な接触時に失敗する傾向があり、RLTがこの弱点を補完する
- RLTは「マルチスケール適応」の一構成要素であり、個別行動の微細な改善、モデルレベルの学習（RECAP）、人間フィードバックによる高レベル推論の適応と組み合わせて使用される

## 2. MEM: Multi-Scale Embodied Memory (Physical Intelligence)

### 2.1 概要

MEM (Multi-Scale Embodied Memory) は、2026年3月にPhysical Intelligence社が発表した手法であり、VLAモデルに**短期視覚メモリと長期言語メモリ**の二重経路メモリシステムを導入することで、最大15分間のタスクを実行可能にする。

### 2.2 アーキテクチャ

MEMは行動予測を以下のように分解する：

```
π(a, l_{t+1}, m_{t+1} | o_{t-T:t}, m_t, g) ≈ π_LL(a | o_{t-K:t}, l_{t+1}, g) × π_HL(l_{t+1}, m_{t+1} | o_t, m_t, g)
```

- **低レベルポリシー (π_LL)**: K個の密な観測を用いて即時アクションを生成
- **高レベルポリシー (π_HL)**: サブタスク指示とメモリの更新を生成

#### 2.2.1 短期メモリ（ビデオエンコーダ）

- **構造**: 修正されたViT（Vision Transformer）に空間-時間分離アテンションを導入
- 空間アテンション：全タイムステップのパッチに対して実行
- **4層ごとに因果的時間アテンション**を挿入：同一空間位置のパッチを全タイムステップにわたって接続し、動きや変化を追跡
- **因果的分離アテンション**により計算量を削減：O(n^2 K^2)（結合アテンション）→ O(Kn^2 + nK^2)
- 上層では過去タイムステップのパッチ表現をドロップし、現在フレームのトークンのみをVLAバックボーンに渡す
- **追加パラメータなし**：事前学習済みViTの重みから直接初期化し、正弦波時間位置エンコーディング（t=0で値0、単一画像と等価）
- **処理能力**: 18フレーム（約54秒）をH100 GPU上でリアルタイム推論制約内で処理可能

#### 2.2.2 長期メモリ（言語ベース）

- **表現形式**: 完了したセマンティックイベントの**自然言語要約** (m_t)
- **更新プロセス**:
  1. 高レベルポリシーが現在の観測、タスク目標、前回のメモリm_tを受信
  2. 次のサブタスク指示 l_{t+1} と更新されたメモリ m_{t+1} を生成
  3. 学習ターゲットはLLMベースのパイプラインで生成：サブタスクアノテーション付きエピソードから、将来のタスク実行に関連する情報を要約
- **圧縮戦略**: LLMに情報の削除・圧縮を促す（例：「薄緑の椀、紺色の椀、明るい黄色の椀」→「3つの椀」）
  - トークン数と推論レイテンシを削減
  - 学習時と推論時の分布シフトを軽減

**重要な知見**: ナイーブな言語メモリ（全サブタスク指示の連結）は大幅に性能が低下する。これは学習時にサブタスクが一度しか出現せず、推論時に失敗の繰り返しで未観測の行動シーケンスが生成されるため。圧縮はサブタスクが成功するまでメモリを更新しないことでこの問題に対処する。

### 2.3 ベースモデルと学習

- **ベースモデル**: π₀.₆ VLA（Gemma 3-4Bプレトレイン済みVLMから初期化）
- **アクション予測**: FASTディスクリートアクショントークン + 860Mパラメータのフローマッチングエキスパート（VLMへの勾配はブロック）
- **入力解像度**: 448×448ピクセル/カメラ、最大4ストリーム
- **事前学習データ**: テレオペレーションデモ、ポリシーロールアウト、人間の修正、ビジョン-言語タスク、**ビデオ-言語タスク**
- **観測シーケンス**: 6フレーム（過去5+現在1）、1秒ストライド → ポストトレーニングで18フレーム、54秒ホライズンに拡張

### 2.4 主要結果

#### ロングホライズンタスク（最大15分）
- **レシピ準備タスク**: MEM ~65-75% vs ベースライン ~20-30%（2-3倍改善）
- **キッチン片付け**: MEM ~55-65% vs ベースライン ~15-25%

#### インコンテキスト適応
- **箸拾い**（テーブル高さ変動）: MEM ~80% vs ベースライン ~30-40%（+62%改善）
- **冷蔵庫開け**（ヒンジ方向不明）: MEM ~70-75% vs ベースライン ~25-30%

#### メモリ能力分析（13タスク）
- **コーヒーすくい**（2杯カウント）: MEM ~80-85%, Pool Memory ~60%, メモリなし ~50%
- **物体探索**（4引き出し）: MEM ~95%, Pool Memory ~60%, メモリなし ~25%
- **食料品開梱**: MEM ~75-80%, Pool Memory ~45%, メモリなし ~35%

#### 器用操作タスク
- MEMはシャツ折り、箱組み立て、テーブル片付け等の非メモリタスクでもπ₀.₆と同等性能を維持（性能劣化なし）

### 2.5 アブレーション

- ビデオメモリ除去: ~25-30%の性能低下
- 言語メモリ除去: ~35-40%の低下（より影響大）
- 言語メモリの圧縮除去（ナイーブ連結）: ~20-25%の低下
- **事前学習の効果**: メモリタスクでMEM事前学習あり ~75% vs ポストトレーニングのみ ~45%（事前学習段階での時間的推論学習が重要）

## 3. VLAに対するRL手法の関連研究

### 3.1 π*₀.₆ / RECAP (Physical Intelligence)

**RECAP (RL with Experience and Corrections via Advantage-conditioned Policies)** は、Physical Intelligenceが開発したオフラインRL事前学習 + オンラインRL自己改善のパイプラインである。

**アーキテクチャ:**
- 5Bパラメータのビジョン-言語モデル + アクションエキスパート
- テキストコマンドや実行品質アノテーションを含むヘテロジニアスプロンプトをサポート

**学習パイプライン（3段階）:**
1. **オフラインRL事前学習**: ベースπ*₀.₆モデルの学習
2. **教師ありファインチューニング**: タスク固有デモンストレーションでの微調整
3. **オンラインRL学習**: ロボット経験 + エキスパート修正での学習

**アドバンテージ条件付け:**
- 「価値の変化」（アドバンテージ）にポリシーを条件付ける
- 実行時には「高アドバンテージのアクションを実行せよ」と指示し、**学習データよりも優れたポリシー**を実現
- 全データ（良いアクションも悪いアクションも）を使用し、どちらが良いかをVLAに伝える

**価値関数と信用割り当て:**
- 特定の状況が他と比べてどれほど良いかを予測するモデルを学習
- ロボットが進捗すると予測値が増加し、進捗がないと横ばい

**エキスパート修正:**
- ロボットが誤った際にテレオペレータが修正を提供
- ポリシーが実際にロボットを置く状況に対する教師信号を提供（複合誤差問題への対処）

**主要結果:**
- エスプレッソ作成: スループットと成功率が2倍以上に向上
- 箱組み立て: ~5 → ~12成功/時間
- 洗濯物畳み: 多様なアイテムで成功率向上
- 最も困難なタスクで**タスクスループットが2倍以上、失敗率が約半分**

### 3.2 πRL: フローベースVLAのオンラインRL (オープンソース)

**πRL**は、フローベースVLA（π₀、π₀.₅）に対するオンラインRLファインチューニングの**初のオープンソースフレームワーク**である。

**核心的課題:**
フローマッチングモデルでは、反復的デノイジングにより**アクション対数尤度が計算不可能**であり、決定論的ODEサンプリングでは探索も不可能。

**技術的解決策:**

**Flow-Noise:**
- デノイジングプロセスに**学習可能なノイズネットワーク**を導入
- 各ステップ遷移が学習可能な分散σ_θ'を持つ等方ガウシアンに
- デノイジングを離散時間MDPとしてモデル化し、デノイジング軌跡全体の結合対数確率から正確なアクション対数尤度を計算
- 1層MDPとして定式化

**Flow-SDE:**
- 決定論的ODEを**等価なSDEに変換**（生成アクションの周辺確率密度を保存）
- 内部デノイジング（τ次元）と外部環境インタラクション（t次元）を結合する**2層MDP**
- ハイブリッドODE-SDEサンプリング：1つのデノイジングステップをランダムにSDE遷移として選択、残りはODE更新 → 計算量約50%削減

**統合方法:**
- VLMパラメータを凍結（メモリ効率化）
- **300Mアクションエキスパートモデルのみファインチューニング**
- PPOでポリシー最適化（クリッピング目的関数）
- 共有Actor-Criticアーキテクチャ

**報酬信号:**
- LIBERO: バイナリ報酬（タスク成功=1、失敗=0）
- ManiSkill: 構造化報酬（正しい配置=1.0、グリッパー取り付け=0.1）
- アクションチャンク全体を1つのマクロステップとして扱い、ステップごとの報酬を合計

**主要結果:**
| ベンチマーク | ベースライン | πRL適用後 |
|------------|------------|---------|
| LIBERO (π₀ few-shot SFT) | 57.6% | 97.6% |
| LIBERO (π₀.₅ few-shot SFT) | 77.1% | 98.3% |
| LIBERO-Long (π₀.₅) | 43.9% | 94.0% |
| ManiSkill (π₀) | 67.2% | 86.7% |
| MultiTask 4352組合せ (π₀) | - | 85.7% |

**PPO vs GRPO**: PPOがGRPOを全LIBEROタスクスイートで一貫して上回る（収束性と安定性で優位）

**制限事項:**
- 学習-評価ギャップ（ODE→SDE変換による）
- 分布外汎化が限定的（VLM凍結のため視覚適応不可）
- 実世界実験なし（シミュレーションのみ）

### 3.3 VLA-RL (オンラインRL for 自己回帰VLA)

**VLA-RL**はOpenVLA-7Bに対するオンラインRLフレームワークである。

**主要技術:**
- ロボットマニピュレーションを「マルチモーダル・マルチターン会話」として定式化
- アクショントークンシーケンスを対話として扱い、トークンレベルの対数確率の合計でアクション対数確率を分解
- PPO + 重要度サンプリング + クリッピングで最適化

**報酬アーキテクチャ:**
- **スパース信号**: バイナリタスク完了フィードバック
- **密信号**: ロボットプロセス報酬モデル（RPRM）による中間報酬
  - RPRMはマイルストーンセグメンテーション（グリッパー状態変化）と進捗ラベリング（エンドエフェクタ速度キーフレーム）から疑似ラベルで学習
- **カリキュラム選択**: 50%のタスク成功率をターゲットとするカリキュラム

**主要結果:**
- LIBEROの40タスクでベースラインを+4.5%改善
- π₀-FASTと同等の性能を達成
- 推論時スケーリングの初期的証拠（テスト時計算量の増加で性能が一貫して向上）

**オフラインRLとの比較:**
- オフライン（SFT）ポリシーはデモンストレーションパターン近傍にアクションが集中
- RL学習ポリシーはアクションがより均一に分布し、分布外状況への適応とロバスト性が向上

### 3.4 その他の重要なRL手法

#### SimpleVLA-RL
- エンドツーエンドのオンラインルールベースRLフレームワーク
- RoboTwin2.0のデュアルアームタスクで**38.3% → 68.8%**（+80%相対改善）
- SFTチューニング済みOpenVLA-OFTで**91% → 99%**

#### VLA-RFT (Reinforcement Fine-Tuning with Verified Rewards)
- データ駆動型ワールドモデルを制御可能なシミュレータとして活用
- わずか**400ファインチューニングステップ**でSFTベースラインを上回る

#### TGRPO (Trajectory-wise GRPO)
- LLMがタスクをサブステージに分解し、密報酬を自動構成
- デュアルレベルアドバンテージ推定（軌跡レベル + ステップレベル）
- Criticネットワーク不要
- LIBEROで平均80.7%の成功率

#### CO-RFT (Chunked Offline RL)
- オフラインRLとオンラインファインチューニングを統合した一貫性ベースの学習目的

#### Test-Time VLA (2026年1月)
- テスト時のRL適応により、多様なVLAバックボーンの性能を未見タスクで一貫して改善
- 分布シフト下でのロバストな汎化と適応性を実証

### 3.5 オンラインRL vs オフラインRLの比較

| 観点 | オフラインRL | オンラインRL |
|------|-----------|-----------|
| データ効率 | 高い（既存データ活用） | 低い（環境インタラクション必要） |
| 分布外性能 | 限定的（訪問状態が制限） | 強い（探索による適応） |
| 中途修正 | 不可能（タスク完了まで待機） | 可能（リアルタイム調整） |
| 実装容易性 | 比較的容易 | シミュレータまたは実機が必要 |
| スケーラビリティ | 高い | 環境並列化が必要 |

**現在のトレンド**: Pre-train → SFT → RL の3段階パラダイムが確立しつつある。

## 4. メモリ・階層的アプローチの関連研究

### 4.1 MemoryVLA (Cognition-Memory-Action Framework)

**MemoryVLA**は、認知科学に着想を得た**知覚-認知メモリバンク（PCMB）**を持つVLAフレームワークである。

**アーキテクチャ:**
- **ビジョン-言語認知モジュール**: 7B Prismatic VLM（並列DINOv2 + SigLIPエンコーダ）
  - SE-bottleneckで256個の知覚トークン (perceptual tokens) に圧縮
  - LLaMA-7BのEOS位置から1個の認知トークン (cognitive token) を抽出
- **知覚-認知メモリバンク**: L個のエントリを保持する2つの並列ストリーム
  - 知覚ストリーム: p ∈ R^(256×dp) - 細粒度の視覚的詳細
  - 認知ストリーム: c ∈ R^(1×dc) - 高レベルセマンティック要約
- **メモリ条件付きアクションエキスパート**: DDIMで10サンプリングステップのDiffusion Transformer、16ステップアクションシーケンス生成

**メモリ検索と融合:**
- 正弦波タイムステップ位置エンコーディングで保存エントリを拡張
- スケールドドット積アテンションで現在のトークンとメモリテンソル間の検索
- **ゲート融合**: g^x = σ(MLP(concat[x, H^x])) により、検索された埋め込みと現在の表現を適応的にブレンド

**メモリ統合（容量超過時）:**
- 隣接エントリ間のコサイン類似度を計算
- 最も類似度が高いペアを平均でマージ

**主要結果:**
- SimplerEnv-Bridge: 71.9%（CogACT-Largeを+14.6上回る）
- LIBERO: 96.5%
- **実世界ロングホライズン: 83%（CogACT比+26）**
- 推論レイテンシ: 0.194秒（ベースライン比+3.6%のオーバーヘッドのみ）

### 4.2 MAP-VLA (Memory-Augmented Prompting)

- 事前学習済みVLAモデルにデモンストレーション由来のメモリプロンプトを付与
- シミュレーションで+7.0%、実ロボットで+25.0%の絶対性能改善

### 4.3 Long-VLA

**Long-VLA**は、ロングホライズンタスク専用に設計された**初のエンドツーエンドVLAモデル**である。

**フェーズアウェア入力マスキング戦略:**
- 各サブタスクを**移動フェーズ**と**インタラクションフェーズ**に分解
- フェーズに関連する感覚入力に選択的にアテンションを向ける
- 入力レベルでの操作（アーキテクチャ変更不要）
- 既存VLAモデルにシームレスに統合可能

**L-CALVINベンチマーク**を提案し、ロングホライズン操作の体系的評価を実現。ベースポリシーおよびπ₀を一貫して上回る。

### 4.4 階層的ポリシーアプローチ

2025-2026年のVLAシステムにおける階層的アプローチは、大きく2つのパラダイムに分類される：

**パラダイム1: VLM + 拡散デコーダ**
- NVIDIA Groot N1、Figure AI Helix
- VLMが高レベルの意思決定、拡散ベースデコーダが低レベルアクション生成

**パラダイム2: 潜在行動語彙**
- LeVERB: ビジョン-言語ポリシーが運動学デモンストレーションから潜在行動語彙を学習、強化学習された制御層が低レベルダイナミクスコマンドを生成

**HiRT (Hierarchical Robot Transformers):**
- VLMが観測をマルチモーダル情報を統合した特徴にエンコード
- 軽量アクションポリシーがこの潜在表現に条件付けて非同期に低レベルアクションを生成
- 高レベルと低レベルの異なる周波数での実行が可能

**GR-RL:**
- 器用かつ精密なロングホライズン操作のためのRL手法
- 高レベル計画と低レベル制御の階層的統合

**WholebodyVLA (ICLR 2026):**
- 全身ロコマニピュレーション制御のための統一潜在VLA

### 4.5 EvoVLA

- **自己進化型VLA**: Long-Horizon Memoryに選択的コンテキストとゲート融合を組み合わせ
- 内在的報酬形成を安定化させ、複雑なロボットタスクに対応

## 5. カスタムVLAシステムへの統合に向けた考察

ユーザの構想（VLA_idea.md）に基づき、各技術の統合可能性を検討する。

### 5.1 構想の整理

```
Stage 1 High Policy (5Hz)
  入力: 短動画クリップ、タスク目標、前回の長期言語メモリ
  バックボーン: ビデオ対応VLM (Qwen3.5 4B)
  出力: 現在のサブタスク、更新された長期言語メモリ

Stage 2 Low Policy (30-100Hz)
  入力: 高ポリシーからのサブタスク指示 + 現在の観測
  出力: 低レベルアクション
```

### 5.2 MEMからの統合要素

**直接適用可能な要素:**
1. **言語ベース長期メモリ**: MEMの自然言語要約メモリ方式は構想のStage 1と完全に一致する。圧縮戦略（LLMによる情報圧縮、成功時のみ更新）を必ず導入すべき。
2. **ビデオエンコーダ設計**: 空間-時間分離アテンション + 上層での過去パッチドロップは、Qwen3.5 4Bの規模でもリアルタイム推論を可能にする設計として参考になる。ViTの重みから直接初期化できる点も有利。
3. **階層的分解**: π_LL × π_HL の分解はそのまま採用可能。

**注意点:**
- MEMの事前学習にはロボット・非ロボットデータの多様な混合が重要（ポストトレーニングのみでは性能45%に留まる）
- 言語メモリのナイーブ連結は避け、圧縮+成功条件付き更新を実装すべき

### 5.3 RLTからの統合要素

**直接適用可能な要素:**
1. **RLトークン抽出**: 高ポリシーVLAから情報ボトルネックトークンを抽出し、小規模Actor-Criticで精密タスクを高速改善するパイプライン
2. **アクション編集アプローチ**: VLAの予測アクションを入力として受け取り「編集」する設計は、ベースモデルの能力を保持しつつ改善する上で有効
3. **実機上学習**: 15分〜2時間での高速学習は実用上大きな利点

**統合提案:**
- 低レベルポリシー（30-100Hz）の精密制御改善にRLTを適用
- 高レベルポリシーはSFTベースで維持し、低レベルのみRLで改善する2段階戦略

### 5.4 RECAPからの統合要素

**アドバンテージ条件付け**は、構想のVLMの言語化能力を活用する上で特に有効：
- 「この行動は良い/悪い」というラベルをVLMの言語入力として与えることで、言語能力を維持しつつRL学習が可能
- エキスパート修正のオンライン統合により、テレオペレーション中にリアルタイムで改善

### 5.5 RL手法の選択指針

| 構想の要件 | 推奨手法 | 理由 |
|----------|---------|------|
| 成功率が高い | RLT + RECAP | 精密制御（RLT）と全体最適化（RECAP）の組み合わせ |
| VLMの言語化能力継承 | RECAP (アドバンテージ条件付け) | 言語プロンプトベースのRL、VLM重み凍結可能 |
| ロングタスク対応 | MEM言語メモリ | 15分以上のタスクに対応可能 |
| 高頻度制御 | RLT | 小規模ネットワークで100Hz実行可能 |
| 研究要素・SOTA | πRL Flow-Noise/SDE | フローベースVLAのRL学習は未開拓領域が多い |

### 5.6 推奨統合アーキテクチャ

```
[高レベルポリシー: 5Hz]
  Qwen3.5 4B + MEMビデオエンコーダ + 言語メモリ
  → サブタスク指示 + メモリ更新
  → RECAPアドバンテージ条件付けで改善可能

[低レベルポリシー: 30-100Hz]
  軽量アクションエキスパート（フローマッチングまたはDiffusion）
  + RLトークン抽出 → 小規模Actor-Critic
  → 精密制御タスクで実機上高速RL学習

[学習パイプライン]
  1. VLM事前学習（ビデオ-言語タスク含む、メモリ能力獲得）
  2. SFT（テレオペレーションデモで全体微調整）
  3. RL Stage 1: RECAP的自己改善（高レベルポリシー）
  4. RL Stage 2: RLT（低レベル精密制御、実機15分〜2時間）
```

### 5.7 C-RADIOv4-SO400Mの活用可能性

構想で言及されているNVIDIA C-RADIOv4は、MEMのビデオエンコーダ設計と組み合わせることで以下の利点が見込まれる：
- 事前学習済み強力な視覚特徴量 → MemoryVLAの知覚トークンとして活用
- SO400Mの効率的な特徴抽出 → リアルタイム推論制約への適合
